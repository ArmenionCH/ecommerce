ARCHITECTURAL PHILOSOPHY
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
Primary directive: EXTENSIBILITY THROUGH MODULAR SEGREGATION.

Build for high initial velocity вҖ” ship working features fast вҖ” while keeping schemas
and folder structures zero-refactor-ready for Phase 2 (vector similarity, ML feeds).

IMMUTABLE BUSINESS RULES (Never Override These)
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
Rule 1 вҖ” COD MANDATE
  All payments are Cash on Delivery. No digital wallets. No escrow. No payment
  gateway integrations. Every order is logged immediately as a pending cash
  collection route. Status: 'placed'.

Rule 2 вҖ” FLAT SHIPPING
  Shipping fee is always exactly вӮұ100.00. No geospatial calculations. No dynamic
  pricing. Hardcoded as SHIPPING_FEE constant in lib/constants.ts.
  Never calculated on the client. Always applied server-side in calculations.ts.

Rule 3 вҖ” ANTI-DEFICIT STOCK LOCK
  The database is the single source of truth for inventory. Client-side state
  cannot override physical stock limits. The trigger tr_enforce_cart_stock
  clamps all cart writes to available stock before they are committed.

Rule 4 вҖ” SOFT DELETE PRESERVATION
  Hard SQL DELETE on products or seller profiles is STRICTLY PROHIBITED.
  Use is_active = false to hide. This protects foreign key integrity across
  historical order receipts, tax records, and analytics.

Rule 5 вҖ” SERVER-SIDE PRICE AUTHORITY
  The checkout engine NEVER reads prices from client payloads. It always
  re-fetches current prices from the database using product_id before writing
  price_at_purchase. This defeats price manipulation attacks.

Rule 6 вҖ” ZERO CLIENT-SIDE DATABASE LOGIC IN ROUTES
  Files inside app/ are pure entry points. Zero raw SQL or Supabase queries.
  All data logic lives in features/*/actions.ts and features/*/hooks/*.

CREATE EXTENSION IF NOT EXISTS pg_trgm; -- Enables trigram text similarity for Phase 2

CREATE TYPE user_role   AS ENUM ('customer', 'seller', 'admin');
CREATE TYPE order_status AS ENUM ('placed', 'packed', 'to_receive', 'received', 'cancelled');

-- PROFILES вҖ” extends auth.users 1:1 via matching UUID
CREATE TABLE public.profiles (
    id               UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY,
    full_name        TEXT NOT NULL,
    role             user_role NOT NULL DEFAULT 'customer',
    phone_number     TEXT,
    delivery_address TEXT,
    metadata         JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at       TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT TIMEZONE('utc', NOW())
);

-- PRODUCTS вҖ” each row owned by a single seller
CREATE TABLE public.products (
    id             BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    seller_id      UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
    title          TEXT NOT NULL,
    description    TEXT,
    price          NUMERIC(10, 2) NOT NULL CONSTRAINT positive_price CHECK (price >= 0.00),
    stock_quantity INT NOT NULL DEFAULT 0 CONSTRAINT non_negative_stock CHECK (stock_quantity >= 0),
    image_url      TEXT,
    is_active      BOOLEAN NOT NULL DEFAULT TRUE,
    created_at     TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT TIMEZONE('utc', NOW())
);

-- PRODUCT VARIATIONS вҖ” size/bundle options per product
CREATE TABLE public.product_variations (
    id             BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    product_id     BIGINT REFERENCES public.products(id) ON DELETE CASCADE NOT NULL,
    name           TEXT NOT NULL,   -- e.g., "Bundle Size"
    value          TEXT NOT NULL,   -- e.g., "500g Pack"
    price_modifier NUMERIC(10, 2) NOT NULL DEFAULT 0.00,
    stock_quantity INT NOT NULL DEFAULT 0 CONSTRAINT var_non_negative_stock CHECK (stock_quantity >= 0)
);

-- CART ITEMS вҖ” active customer baskets, DB-enforced quantities
CREATE TABLE public.cart_items (
    id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    customer_id  UUID REFERENCES public.profiles(id) ON DELETE CASCADE NOT NULL,
    product_id   BIGINT REFERENCES public.products(id) ON DELETE CASCADE NOT NULL,
    variation_id BIGINT REFERENCES public.product_variations(id) ON DELETE SET NULL,
    quantity     INT NOT NULL DEFAULT 1 CONSTRAINT positive_quantity CHECK (quantity > 0),
    created_at   TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT TIMEZONE('utc', NOW()),
    UNIQUE(customer_id, product_id, variation_id)
);

-- ORDERS вҖ” master COD order records
CREATE TABLE public.orders (
    id               BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    customer_id      UUID REFERENCES public.profiles(id) ON DELETE SET NULL NOT NULL,
    total_amount     NUMERIC(10, 2) NOT NULL CONSTRAINT positive_total CHECK (total_amount >= 0.00),
    status           order_status NOT NULL DEFAULT 'placed',
    shipping_address TEXT NOT NULL,  -- Snapshot at time of purchase; never changes
    created_at       TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT TIMEZONE('utc', NOW())
);

-- ORDER ITEMS вҖ” immutable line-item receipts (price_at_purchase never changes)
CREATE TABLE public.order_items (
    id                BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_id          BIGINT REFERENCES public.orders(id) ON DELETE CASCADE NOT NULL,
    product_id        BIGINT REFERENCES public.products(id) ON DELETE SET NULL NOT NULL,
    seller_id         UUID REFERENCES public.profiles(id) NOT NULL,
    variation_details TEXT,  -- e.g., "Size: XL" вҖ” string snapshot, not FK
    quantity          INT NOT NULL DEFAULT 1 CONSTRAINT item_positive_quantity CHECK (quantity > 0),
    price_at_purchase NUMERIC(10, 2) NOT NULL CONSTRAINT target_price_snapshot CHECK (price_at_purchase >= 0.00)
);

-- REVIEWS вҖ” one review per customer per product; only buyers can review
CREATE TABLE public.reviews (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    product_id  BIGINT REFERENCES public.products(id) ON DELETE CASCADE NOT NULL,
    customer_id UUID REFERENCES public.profiles(id) ON DELETE SET NULL NOT NULL,
    rating      INT NOT NULL CONSTRAINT star_rating_bounds CHECK (rating >= 1 AND rating <= 5),
    comment     TEXT,
    created_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT TIMEZONE('utc', NOW()),
    UNIQUE(product_id, customer_id)  -- One review per buyer per product
);

-- TRIGGER A: Clamp cart quantities to available stock (before write)
CREATE OR REPLACE FUNCTION public.check_cart_stock_limits()
RETURNS TRIGGER
LANGUAGE plpgsql AS $$
DECLARE
    v_available_stock INT;
BEGIN
    -- Read current live stock from the database (not from client)
    SELECT stock_quantity
    INTO   v_available_stock
    FROM   public.products
    WHERE  id = NEW.product_id;

    -- Clamp requested quantity to available stock
    IF NEW.quantity > v_available_stock THEN
        NEW.quantity := v_available_stock;
    END IF;

    -- Abort silently if stock is completely depleted
    IF NEW.quantity = 0 THEN
        RETURN NULL;
    END IF;

    RETURN NEW;
END;
$$;

CREATE TRIGGER tr_enforce_cart_stock
BEFORE INSERT OR UPDATE ON public.cart_items
FOR EACH ROW EXECUTE FUNCTION public.check_cart_stock_limits();

-- TRIGGER B: Deduct stock from products table after order_item is written
CREATE OR REPLACE FUNCTION public.deduct_product_stock()
RETURNS TRIGGER
LANGUAGE plpgsql AS $$
BEGIN
    UPDATE public.products
    SET    stock_quantity = stock_quantity - NEW.quantity
    WHERE  id = NEW.product_id;
    -- Note: non_negative_stock constraint will abort if stock would go below 0
    RETURN NEW;
END;
$$;

CREATE TRIGGER tr_deduct_stock
AFTER INSERT ON public.order_items
FOR EACH ROW EXECUTE FUNCTION public.deduct_product_stock();

-- AUTO-CREATE PROFILE TRIGGER: fires after every new auth.users row
CREATE OR REPLACE FUNCTION public.handle_new_user_registration()
RETURNS TRIGGER
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public AS $$
BEGIN
    INSERT INTO public.profiles (id, full_name, role)
    VALUES (
        NEW.id,
        COALESCE(NEW.raw_user_meta_data ->> 'full_name', 'New User'),
        COALESCE((NEW.raw_user_meta_data ->> 'role')::user_role, 'customer')
    );
    RETURN NEW;
END;
$$;

CREATE TRIGGER tr_on_auth_user_created
AFTER INSERT ON auth.users
FOR EACH ROW EXECUTE FUNCTION public.handle_new_user_registration();

-- SELLER ANALYTICS VIEW вҖ” reads from order_items + orders; no frontend math needed
CREATE OR REPLACE VIEW public.seller_analytics AS
SELECT
    oi.seller_id,
    COALESCE(SUM(oi.price_at_purchase * oi.quantity), 0.00)          AS total_earnings,
    COUNT(DISTINCT oi.order_id)                                        AS total_orders_handled,
    COUNT(DISTINCT CASE WHEN o.status = 'placed' THEN o.id END)       AS pending_orders_count,
    COUNT(DISTINCT CASE WHEN o.status = 'received' THEN o.id END)     AS completed_orders_count
FROM   public.order_items oi
JOIN   public.orders o ON oi.order_id = o.id
WHERE  o.status != 'cancelled'
GROUP BY oi.seller_id;

    ON public.products FOR INSERT
    WITH CHECK (auth.get_user_role() = 'seller' AND auth.uid() = seller_id);

    ON public.products FOR UPDATE
    USING (auth.uid() = seller_id OR auth.get_user_role() = 'admin');

    ON public.product_variations FOR SELECT USING (true);

    ON public.product_variations FOR ALL
    USING (
        auth.get_user_role() = 'admin'
        OR EXISTS (
            SELECT 1 FROM public.products p
            WHERE p.id = product_id AND p.seller_id = auth.uid()
        )
    );

    ON public.cart_items FOR SELECT USING (auth.uid() = customer_id);

    ON public.cart_items FOR ALL USING (auth.uid() = customer_id);

    ON public.orders FOR SELECT
    USING (
        auth.uid() = customer_id
        OR auth.get_user_role() = 'admin'
        OR EXISTS (
            SELECT 1 FROM public.order_items oi
            WHERE oi.order_id = id AND oi.seller_id = auth.uid()
        )
    );

CREATE POLICY "orders_insert_customer_only"
    ON public.orders FOR INSERT
    WITH CHECK (auth.uid() = customer_id);

CREATE POLICY "orders_update_seller_or_admin"
    ON public.orders FOR UPDATE
    USING (
        auth.get_user_role() = 'admin'
        OR EXISTS (
            SELECT 1 FROM public.order_items oi
            WHERE oi.order_id = id AND oi.seller_id = auth.uid()
        )
    );

-- ORDER ITEMS POLICIES
CREATE POLICY "order_items_select_related_parties"
    ON public.order_items FOR SELECT
    USING (
        seller_id = auth.uid()
        OR auth.get_user_role() = 'admin'
        OR EXISTS (
            SELECT 1 FROM public.orders o
            WHERE o.id = order_id AND o.customer_id = auth.uid()
        )
    );

CREATE POLICY "order_items_insert_system_only"
    ON public.order_items FOR INSERT
    WITH CHECK (auth.uid() IS NOT NULL);

-- REVIEWS POLICIES
CREATE POLICY "reviews_select_public"
    ON public.reviews FOR SELECT USING (true);

CREATE POLICY "reviews_insert_verified_buyers_only"
    ON public.reviews FOR INSERT
    WITH CHECK (
        auth.uid() = customer_id
        AND EXISTS (
            SELECT 1
            FROM   public.orders o
            JOIN   public.order_items oi ON oi.order_id = o.id
            WHERE  o.customer_id = auth.uid()
            AND    oi.product_id  = product_id
            AND    o.status       = 'received'
        )
    );

/**
 * SERVER-SIDE ORDER CALCULATION ENGINE
 *
 * SECURITY CONTRACT:
 *   - This function NEVER accepts prices from the client.
 *   - It re-fetches every product price from the database using product_id.
 *   - This defeats price-manipulation attacks entirely.
 *   - Must only be called from server actions (features/checkout/actions.ts).
 */

import { createServerClient }      from '@/lib/supabaseServer';
import { SHIPPING_FEE }            from '@/lib/constants';
import type { OrderPlacementPayload } from '@/lib/types';

export interface CalculatedLine {
  product_id        : number;
  seller_id         : string;
  variation_id      : number | null;
  quantity          : number;
  price_at_purchase : number;  // From DB вҖ” not from client
  variation_details?: string;
}

export interface CalculationResult {
  lines        : CalculatedLine[];
  subtotal     : number;
  shippingFee  : number;
  grandTotal   : number;
}

/**
 * Fetches authoritative prices from DB and computes the final order total.
 * @param payload - Order payload from the client (prices intentionally excluded)
 * @returns Calculated totals and immutable line items
 */
export async function calculateOrderTotal(
  payload: OrderPlacementPayload
): Promise<CalculationResult> {
  const supabase = createServerClient();

  const productIds = payload.items.map(item => item.product_id);

  // Fetch authoritative prices in a single query
  const { data: products, error } = await supabase
    .from('products')
    .select('id, price, title')
    .in('id', productIds)
    .eq('is_active', true);

  if (error || !products) {
    throw new Error(`Price fetch failed: ${error?.message ?? 'No products returned'}`);
  }

  // Map product prices for O(1) lookup
  const priceMap = new Map<number, number>(
    products.map(p => [p.id, p.price])
  );

  let subtotal = 0;

  const lines: CalculatedLine[] = payload.items.map(item => {
    const dbPrice = priceMap.get(item.product_id);

    if (dbPrice === undefined) {
      throw new Error(`Product ${item.product_id} not found or inactive.`);
    }

    const lineTotal = dbPrice * item.quantity;
    subtotal += lineTotal;

    return {
      product_id        : item.product_id,
      seller_id         : item.seller_id,
      variation_id      : item.variation_id,
      quantity          : item.quantity,
      price_at_purchase : dbPrice,
    };
  });

  return {
    lines,
    subtotal,
    shippingFee : SHIPPING_FEE,
    grandTotal  : subtotal + SHIPPING_FEE,
  };
}

/**
 * ORDER PLACEMENT SERVER ACTION
 *
 * Execution sequence (must be atomic):
 *   1. Re-fetch authoritative prices via calculateOrderTotal()
 *   2. Insert master order row into public.orders
 *   3. Insert itemized rows into public.order_items
 *      вҶ’ Triggers tr_deduct_stock automatically (DB trigger)
 *   4. Clear customer's cart_items from the database
 *   5. Return success + orderId to client
 */

import { createServerClient }                           from '@/lib/supabaseServer';
import { calculateOrderTotal }                          from './utils/calculations';
import type { OrderPlacementPayload, OrderPlacementResult } from '@/lib/types';

export async function executeOrderPlacement(
  payload: OrderPlacementPayload
): Promise<OrderPlacementResult> {
  const supabase = createServerClient();

  try {
    // Step 1: Calculate totals using DB prices (never client prices)
    const { lines, grandTotal } = await calculateOrderTotal(payload);

    // Step 2: Write master order record
    const { data: orderRow, error: orderError } = await supabase
      .from('orders')
      .insert({
        customer_id     : payload.customerId,
        total_amount    : grandTotal,
        status          : 'placed',
        shipping_address: payload.shippingAddress,
      })
      .select('id')
      .single();

    if (orderError || !orderRow) {
      throw new Error(`Order creation failed: ${orderError?.message}`);
    }

    const orderId = orderRow.id;

    // Step 3: Write itemized order_items (triggers tr_deduct_stock per row)
    const orderItemRows = lines.map(line => ({
      order_id          : orderId,
      product_id        : line.product_id,
      seller_id         : line.seller_id,
      variation_id      : line.variation_id,
      quantity          : line.quantity,
      price_at_purchase : line.price_at_purchase,
    }));

    const { error: itemsError } = await supabase
      .from('order_items')
      .insert(orderItemRows);

    if (itemsError) {
      throw new Error(`Order items write failed: ${itemsError.message}`);
    }

    // Step 4: Clear customer's active cart
    const { error: cartError } = await supabase
      .from('cart_items')
      .delete()
      .eq('customer_id', payload.customerId);

    if (cartError) {
      // Non-fatal вҖ” order is placed; log and continue
      console.warn('[GreenMarket] Cart clear failed (non-fatal):', cartError.message);
    }

    return { success: true, orderId };

  } catch (err) {
    const message = err instanceof Error ? err.message : 'Unknown error';
    console.error('[GreenMarket] executeOrderPlacement error:', message);
    return { success: false, error: message };
  }
}

/**
 * CART ACTIONS HOOK
 *
 * All cart mutations go through this hook. QuantityInput.tsx must
 * NEVER directly access Supabase or session state вҖ” it calls this hook.
 * The DB trigger tr_enforce_cart_stock handles quantity clamping.
 */

import { useState, useEffect, useCallback } from 'react';
import { supabaseClient }                   from '@/lib/supabase';
import type { CartItem }                    from '@/lib/types';

interface UseCartActionsReturn {
  cartItems   : CartItem[];
  isLoading   : boolean;
  addItem     : (productId: number, variationId: number | null, qty: number) => Promise<void>;
  removeItem  : (cartItemId: number) => Promise<void>;
  updateQty   : (cartItemId: number, newQty: number) => Promise<void>;
  clearCart   : () => Promise<void>;
}

export function useCartActions(customerId: string | null): UseCartActionsReturn {
  const [cartItems, setCartItems] = useState<CartItem[]>([]);
  const [isLoading, setLoading]   = useState<boolean>(true);

  // Load cart items with product join
  const loadCart = useCallback(async () => {
    if (!customerId) { setCartItems([]); setLoading(false); return; }

    const { data } = await supabaseClient
      .from('cart_items')
      .select('*, product:products(*), variation:product_variations(*)')
      .eq('customer_id', customerId)
      .order('created_at', { ascending: true });

    setCartItems((data as CartItem[]) ?? []);
    setLoading(false);
  }, [customerId]);

  useEffect(() => { loadCart(); }, [loadCart]);

  const addItem = async (productId: number, variationId: number | null, qty: number) => {
    if (!customerId) return;
    // Upsert вҖ” DB trigger will clamp quantity to available stock
    await supabaseClient
      .from('cart_items')
      .upsert({
        customer_id : customerId,
        product_id  : productId,
        variation_id: variationId,
        quantity    : qty,
      }, { onConflict: 'customer_id,product_id,variation_id' });

    await loadCart();
  };

  const removeItem = async (cartItemId: number) => {
    await supabaseClient.from('cart_items').delete().eq('id', cartItemId);
    await loadCart();
  };

  const updateQty = async (cartItemId: number, newQty: number) => {
    if (newQty <= 0) { await removeItem(cartItemId); return; }
    // DB trigger clamps to max available stock
    await supabaseClient
      .from('cart_items')
      .update({ quantity: newQty })
      .eq('id', cartItemId);
    await loadCart();
  };

  const clearCart = async () => {
    if (!customerId) return;
    await supabaseClient.from('cart_items').delete().eq('customer_id', customerId);
    setCartItems([]);
  };

  return { cartItems, isLoading, addItem, removeItem, updateQty, clearCart };
}

EDGE CASE A вҖ” PRICE MANIPULATION ATTACK
  Threat    : Customer intercepts checkout payload, submits price = вӮұ1.00
  Defense   : calculateOrderTotal() ignores client prices entirely.
              Uses product_id to re-fetch price from public.products.
              price_at_purchase is written from DB value вҖ” never from client.
  Test      : features/testing/systemIntegration.test.ts вҖ” Test 3

EDGE CASE B вҖ” CONCURRENT CHECKOUT RACE CONDITION (Last-Item Over-Sell)
  Threat    : Two customers check out the last item simultaneously.
  Defense   : PostgreSQL serializes writes to order_items.
              tr_deduct_stock fires instantly on first commit.
              Second commit hits non_negative_stock CHECK constraint.
              DB rejects second order and returns error to second user.
              No inventory corruption. No double-sell.

EDGE CASE C вҖ” CART QUANTITY INJECTION (Client-Side Manipulation)
  Threat    : Customer edits browser state to set cart quantity to 99999.
  Defense   : tr_enforce_cart_stock fires BEFORE INSERT OR UPDATE.
              Reads live stock_quantity from DB.
              Clamps to max available. Client-submitted value is discarded.

EDGE CASE D вҖ” STALE CART PRICE (Price Changed After Add-to-Cart)
  Threat    : Item was вӮұ500 when added to cart. Seller changes it to вӮұ800.
              Customer checks out expecting to pay вӮұ500.
  Defense   : calculateOrderTotal() re-fetches current prices at checkout time.
              Customer pays вӮұ800 (the current price).
              Future improvement: show price-change warning in cart UI.

EDGE CASE E вҖ” SELLER SOFT DELETE (Historical Order Integrity)
  Threat    : Admin hard-deletes a seller account. Breaks order_items FK.
  Defense   : Hard DELETE is prohibited by architecture rules.
              Sellers are soft-hidden (is_active = false) only.
              order_items.seller_id FK is preserved.
              Historical receipts remain queryable forever.

EDGE CASE F вҖ” REVIEW BOMBING (Non-Buyer Reviews)
  Threat    : Anyone submits fake 1-star reviews on competitor products.
  Defense   : RLS policy on reviews checks for matching order with status = 'received'.
              Non-buyers get a policy violation error at the DB layer.
              Feature is server-enforced вҖ” not just client validation.

                       PART IX: CODING STYLE GUIDE (Human Maintainability)
в”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғв”Ғ

These rules apply to every file in this codebase. Follow them for consistent,
human-readable, AI-agent-resumable code.

1. IMPORTS: Always use path aliases. Never use relative paths deeper than 1 level.
   вң…  import { supabaseClient } from '@/lib/supabase';
   вқҢ  import { supabaseClient } from '../../../lib/supabase';

2. INTERFACES: Always use interfaces for object shapes. Export from lib/types.ts.
   Never redeclare types inline across multiple files.

3. ALIGNMENT: Align object properties and variable assignments vertically.
   This makes diffs readable and errors obvious at a glance.
   вң…  const result = {
         id     : 1,
         name   : 'hello',
         active : true,
       };

4. COMMENTS: Every function exported from features/ must have a JSDoc block:
   /** One-line description of what it does */
   @param, @returns if non-obvious.
   Include SECURITY CONTRACT notes on any function that handles money or auth.

5. NAMING CONVENTIONS:
   - Components   : PascalCase.tsx           (ProductCard.tsx)
   - Hooks        : camelCase, use prefix    (useCartActions.ts)
   - Actions      : camelCase, verb prefix   (executeOrderPlacement, submitReview)
   - Constants    : SCREAMING_SNAKE_CASE     (SHIPPING_FEE)
   - Types/Ifaces : PascalCase               (OrderItem, UserSession)
   - DB columns   : snake_case               (customer_id, created_at)

6. SERVER ACTIONS: All files using 'use server' must have that directive on line 1.
   All files using 'use client' must have that directive on line 1.
   Never mix server and client code in the same file.

7. ERROR HANDLING: Every async function must have try/catch.
   Return { success: boolean, error?: string } from server actions.
   Never let errors propagate silently to the UI.

8. NO RAW SQL IN ROUTES: app/*.tsx files call feature components only.
   Features call actions.ts or hooks. Never bypass this layering.

9. SOFT DELETES ONLY: Never write .delete() on products or profiles in production.
   Use .update({ is_active: false }) instead. See Rule 4 in Part I.

10. PRICE CALCULATIONS: The word "price" from a client payload is never trusted.
    If you are writing a checkout or order function and reading payload.price,
    stop. Re-fetch from the database using the product_id instead.
